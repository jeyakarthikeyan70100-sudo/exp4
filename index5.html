Dijkstra's Algorithm

Dijkstra's Algorithm is a popular greedy algorithm used to find the shortest path from a single source vertex to all other vertices in a weighted graph with non-negative edge weights.

Run Interactive Demo View Python Code

What is Dijkstra's Algorithm?

Dijkstra's Algorithm solves the Single Source Shortest Path problem by calculating the minimum distance from one selected source vertex to every other vertex in a weighted graph.

Single Source

The algorithm starts from one source vertex and calculates the shortest distance to all other vertices.

Weighted Graph

It works on weighted graphs where every edge has a non-negative numerical cost or distance.

Shortest Path

It determines the minimum-cost path from the source vertex to every reachable vertex.

How Dijkstra's Algorithm Works

The algorithm repeatedly selects the unvisited vertex with the smallest known distance and relaxes its neighboring edges.

1
Initialize Distances

Set the source distance to 0 and all other vertex distances to infinity.

2
Select Minimum Vertex

Select the unvisited vertex with the smallest tentative distance.

3
Relax Edges

Check all neighboring vertices. Update their distances if a shorter path is found.

4
Mark as Visited

Mark the selected vertex as visited so it will not be processed again.

5
Repeat

Continue until all reachable vertices have been processed.

Key Concept: Relaxation

For an edge from vertex u to vertex v with weight w, Dijkstra's algorithm checks whether the following condition is true:

distance[v] > distance[u] + weight(u,v)

If the condition is true, the distance of vertex v is updated with the new shorter distance.

This process is called edge relaxation.

Python Implementation

The following Python program implements Dijkstra's Single Source Shortest Path Algorithm using a priority queue. 

import heapq

def dijkstra(graph, source):

    # Set initial distances to infinity
    distances = {
        vertex: float('inf')
        for vertex in graph
    }

    # Distance from source to itself is 0
    distances[source] = 0

    # Priority queue
    priority_queue = [(0, source)]

    while priority_queue:

        # Get vertex with minimum distance
        current_distance, current_vertex = \
            heapq.heappop(priority_queue)

        # Ignore outdated entries
        if current_distance > distances[current_vertex]:
            continue

        # Visit all neighboring vertices
        for neighbor, weight in graph[current_vertex]:

            distance = current_distance + weight

            # Relaxation
            if distance < distances[neighbor]:

                distances[neighbor] = distance

                heapq.heappush(
                    priority_queue,
                    (distance, neighbor)
                )

    return distances

Time and Space Complexity

The complexity depends on the data structure used to implement the priority queue.

Binary Heap

O((V + E) log V)

Efficient implementation using a priority queue.

Array

O(V²)

Simple implementation using an array to find minimum distance.

Space Complexity

O(V + E)

Space required for graph, distances and priority queue.

Applications of Dijkstra's Algorithm

Dijkstra's Algorithm is widely used in computer science and real-world systems.

GPS Navigation

Finding shortest routes between locations.

Computer Networks

Finding efficient routes for data packets.

Games

Pathfinding for characters and game objects.

Transportation

Optimizing routes in transportation systems.

Advantages

  • Finds shortest paths efficiently.
  • Works well with non-negative edge weights.
  • Can calculate shortest paths to all vertices.
  • Useful in routing and navigation systems.

Limitation

  • Does not work correctly with negative edge weights.
  • Negative weight edges require algorithms such as Bellman-Ford.
  • The algorithm assumes the graph has non-negative edge costs.

Conclusion

Dijkstra's Algorithm is one of the most important graph algorithms used to solve the Single Source Shortest Path problem. It uses a greedy approach to repeatedly select the closest unvisited vertex and relax its neighboring edges. With a priority queue, the algorithm provides an efficient solution for graphs containing non-negative edge weights.
